# 亲密表达与边界状态评估：测量逻辑说明

## 1. 目的

- 识别用户当前在亲密表达中的负担、阻滞和可用资源
- 优先帮助用户理解“哪里更费力、哪里更需要关注”，而不是给出固定人格结论
- 为结果页提供稳定、可解释、可降敏的生成规则

## 2. 结果哲学

- 结果以“当前状态”表述，不以“人格类型”表述
- 结果用于自我理解，不用于诊断、评价他人或判断关系对错
- 输出优先级固定为：
  1. 状态摘要
  2. 需要优先关注的信号
  3. 维度结果
  4. 行动建议
  5. 次级状态模式
- 结果文案必须使用：
  - 当前
  - 倾向
  - 可能
  - 更接近
- 结果文案不得使用：
  - 异常
  - 病态
  - 障碍
  - 你就是
  - 你属于固定人格

## 3. 三层测量模型

### 3.1 核心状态维度

用于识别当前主要负担来源。

- 表达负担
  - 关注表达前后的回避、压抑、犹豫、后悔和内耗
- 羞耻负担
  - 关注对亲密需求、幻想、表达本身的羞耻、自责与自我否定
- 关系安全感
  - 关注表达后被误解、拒绝、疏远的担忧

### 3.2 保护性维度

用于识别当前可用资源，不直接折叠为风险反面。

- 沟通开放度
  - 是否能进入对话，讨论偏好、边界、不适和需要
- 自我接纳
  - 是否能接住自己的差异、节奏和不确定感
- 边界表达能力
  - 是否能在当下识别不适并明确表达

### 3.3 风险提示维度

用于单独触发警示，不并入普通维度总分。

- 边界受压
  - 例如顺着继续、事后才意识到不愿意、难以及时停下
- 持续内耗
  - 例如明显压抑后反复反刍、自责或长期消耗
- 明显回避 / 冻结
  - 例如有必要也尽量不谈、面对相关话题明显卡住

## 4. 作答口径

- 优先根据最近 3 个月，或最近一段典型亲密关系中的状态作答
- 当前无关系经历时，可根据自己面对亲密互动的一般反应作答
- 题目时间口径统一使用：
  - 过去一段时间里
  - 在关系中
  - 通常

## 5. 题目设计原则

- 优先测负担、成本、回避、受损感，不优先测风格差异
- 避免把“保守 / 不主动”直接等同为问题
- 尽量使用可观察行为或可识别体验
- 避免模糊副词，如：
  - 比较自然
  - 较清楚
  - 较早察觉
- 每题只测一个判断
- 不使用双重否定
- 不使用过度露骨表达

## 6. 选项与有效题规则

### 6.1 标准选项

- 非常不同意 = 1
- 不同意 = 2
- 一般 = 3
- 同意 = 4
- 非常同意 = 5
- 不适用 / 没有经历 / 无法判断 = `NA`

### 6.2 `NA` 处理

- `NA` 不计入任何分数
- 维度有效题数不足时，不输出该维度强结论
- 推荐阈值：
  - 5 题维度至少回答 3 题
  - 少于 3 题时展示“信息不足”

## 7. 反向题规则

- 不按维度强制配额
- 总量控制在总题量约 20%
- 仅保留确实能提高辨别度且可读性良好的反向题
- 反向计分规则：
  - 1 -> 5
  - 2 -> 4
  - 3 -> 3
  - 4 -> 2
  - 5 -> 1

## 8. 评分原则

### 8.1 维度分

- 每个维度独立计算
- 维度分使用有效题平均值转换到 0-100

公式：

`维度分 = ((平均分 - 1) / 4) * 100`

### 8.2 结果使用原则

- 不输出“总压抑指数”作为主结果
- 不把保护性维度简单反向并入风险总分
- 状态摘要由“高负担 + 低保护”组合生成
- 风险提示由风险触发逻辑独立生成

### 8.3 内部辅助排序字段

如果实现需要内部排序字段，可保留一个启发式总览分，但必须满足：

- 不对用户展示为主结论
- 不用于生成病理化等级
- 文档中明确标注为内部启发式字段

## 9. 风险触发规则

- 风险触发逻辑与普通维度逻辑分离
- 风险触发题可以来自不同维度
- 一旦达到阈值，结果页必须优先展示“需要优先关注的信号”
- 风险触发后：
  - 弱化次级状态模式
  - 弱化分享入口
  - 强化自我保护、边界与支持性建议
  - 重复非替代支持提示

## 10. 结果结构

### 10.1 当前状态摘要

由高负担维度与低保护维度组合生成，示例：

- 当前表达负担偏高，且关系安全感偏低
- 当前羞耻负担偏高，自我接纳资源偏弱
- 当前边界表达能力偏弱，建议优先关注关系中的不适信号

### 10.2 需要优先关注的信号

仅在风险触发时出现，示例：

- 过去一段时间里，你可能更容易在不舒服的情况下继续配合
- 你当前可能承受较明显的关系内耗或表达冻结

### 10.3 维度结果

- 每个维度独立展示
- 使用中性等级：
  - 较低
  - 偏低
  - 中等
  - 偏高
  - 较高
- 有效题不足时显示“信息不足”

### 10.4 行动建议

- 先根据风险触发输出
- 再根据高负担维度补充
- 最后根据低保护维度补资源建议

### 10.5 次级状态模式

- 仅作辅助理解
- 不使用人格化命名
- 示例：
  - 当前表达负担偏高
  - 当前关系警觉偏高
  - 当前边界表达受阻

## 11. 文案边界

- 不写“你有问题”
- 不写“你就是某种人”
- 不写“你的关系一定如何”
- 强调：
  - 当前自评结果
  - 仅反映最近状态
  - 可随关系情境和支持资源变化

## 12. 结果安全性要求

- 高风险结果不能娱乐化
- 分享内容默认去敏
- 结果不应鼓励用户把结论套用到伴侣身上
- 结果页必须提醒：
  - 建议在私密设备上查看
  - 不建议将结果用于评判他人
  - 不替代专业支持
