# 亲密表达与边界状态评估 MVP PRD

## 1. 基本信息

- 项目名：亲密表达与边界状态评估
- 阶段：MVP
- 形态：前端单页测试功能
- 范围：不登录、无后端、无云端存储、仅轻分享
- 题量：30 题
- 核心目标：帮助用户理解当前的表达负担、边界状态、关系安全感、自我接纳与沟通开放水平
- 关联文档：
  - [测量逻辑说明](./intimacy-test-measurement-spec.md)
  - [题库与结果规则表](./intimacy-test-rules.md)

## 2. 产品目标

### 2.1 核心目标

- 提供完整可运行的测试链路：入口、答题、结果、轻分享
- 以当前状态理解为主，不以人格标签为主
- 结果输出优先级固定为：
  1. 当前状态摘要
  2. 需要优先关注的信号
  3. 维度结果
  4. 行动建议
  5. 次级状态模式
- 支持本地恢复进度与最近一次结果

### 2.2 非目标

- 不做医学或心理诊断
- 不做账号体系
- 不做服务端存储
- 不做公开结果链接
- 不做双人测试
- 不做关系匹配或伴侣评判工具

## 3. 适用范围

- 仅面向成年人
- 优先根据最近 3 个月，或最近一段典型亲密关系中的状态作答
- 当前无关系经历时，可根据自己对亲密互动的一般反应作答
- 本测试仅用于自我理解，不替代心理咨询、医学建议或危机支持

## 4. 用户价值

- 理解自己当前在哪些环节承担较高的表达负担
- 识别需要优先关注的边界、安全感或持续内耗信号
- 看到自己的保护性资源，如沟通开放度、自我接纳、边界表达能力
- 获得更具体的下一步建议

## 5. MVP 范围

### 5.1 包含

- 测试入口页
- 30 题答题流程
- 本地结果计算
- 结果报告页
- 中性轻分享卡
- 本地进度恢复
- 题库版本隔离

### 5.2 不包含

- 登录
- 后端接口
- 公共结果链接
- 历史测试对比
- 社交互动
- AI 诊断或扩写

## 6. 用户流程

1. 用户进入测试入口页
2. 阅读适用范围、作答口径、隐私说明、免责声明
3. 点击开始测试
4. 完成 30 道题；允许选择“不适用 / 没有经历 / 无法判断”
5. 前端本地生成结果
6. 结果页先展示状态摘要与风险提示，再展示维度结果与建议
7. 用户可选择轻分享、重新测试或清空结果

## 7. 页面结构

### 7.1 测试入口页

内容：

- 标题：亲密表达与边界状态评估
- 副标题：用于理解当前的表达负担、关系安全感与边界状态
- 预计耗时
- 题量说明
- 适用范围
- 作答口径
- 隐私说明
- 风险与支持说明
- 开始按钮

### 7.2 答题页

内容：

- 当前题号 / 总题数
- 进度条
- 题目正文
- 6 个选项：
  - 非常不同意
  - 不同意
  - 一般
  - 同意
  - 非常同意
  - 不适用 / 没有经历 / 无法判断
- 上一题 / 下一题
- 最后一题显示提交按钮

要求：

- 自动保存作答状态
- 支持刷新恢复
- 支持返回修改
- 不展示总分或实时结论

### 7.3 结果报告页

结果顺序固定为：

1. 当前状态摘要
2. 需要优先关注的信号
3. 维度结果
4. 行动建议
5. 次级状态模式

内容：

- 当前状态摘要
- 需要优先关注的信号
- 六维度结果卡
- 结果说明
- 行动建议
- 次级状态模式
- 隐私提醒
- 重新测试按钮
- 轻分享按钮

约束：

- 不输出强人格化画像
- 不输出病理化措辞
- 高风险结果优先展示风险模块，并弱化分享入口

### 7.4 轻分享弹层或分享页

内容：

- 测试名称
- 中性状态摘要
- 1-2 个保护性亮点或建议方向
- 产品入口提示

约束：

- 不展示风险标签
- 不展示敏感等级
- 不展示详细题目与详细维度
- 高风险结果默认仅允许极简中性卡片

## 8. 隐私与安全策略

- 所有答案仅本地存储
- 不上传服务端
- 建议在私密设备上作答
- 不建议将结果用于评判伴侣或他人
- 不建议将结果作为关系对错依据

## 9. 验收标准

### 9.1 功能验收

- 用户可完成 30 题测试，并在刷新后恢复进度
- 支持“不适用”选项
- 可稳定生成结果页
- 可生成轻分享卡
- 可重置并重新测试

### 9.2 内容安全验收

- 对外标题、分层、结果文案不使用病理化或强定性措辞
- 结果文案统一使用“当前状态 / 倾向 / 可能”
- 高风险结果优先展示风险提示
- 高风险结果分享内容默认降敏
- 不把复杂状态压缩为唯一人格结论

### 9.3 规则验收

- 维度、风险、分享策略均配置化
- 评分逻辑与风险逻辑相互独立
- 题库版本隔离
- 不同题库版本不会复用旧答案

## 10. 开发顺序

1. 确认产品命名、适用范围、风险处理原则
2. 确认测量逻辑和结果输出哲学
3. 确认题库与规则表
4. 实现答题流程、本地存储与结果页
5. 实现轻分享与降敏策略
